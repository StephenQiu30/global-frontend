# PRD 10: 公开仓库 URL 预览翻译后续能力

## 1. 目标

后续支持用户输入任意公开 GitHub 仓库 URL，系统可以读取公开 Markdown 文件并生成翻译预览。提交回 GitHub 时再引导安装 GitHub App 或使用 fork + PR 流程。

## 2. 范围

这是第二阶段能力，不进入第一版实现。

第二阶段支持：

1. 公开仓库 URL 解析。
2. 无安装状态下读取公开 Markdown。
3. 翻译预览。
4. 下载翻译文件。
5. 引导安装 GitHub App 后提交 PR。

第二阶段可选：

1. fork 仓库。
2. 从 fork 向原仓库创建 PR。
3. OAuth 用户授权。

## 3. 用户故事

作为普通用户，我希望输入任意公开仓库地址就能预览 README 翻译，这样我可以先确认翻译效果，再决定是否安装 GitHub App 或提交 PR。

## 4. 第一版为什么不做

1. 提交回 GitHub 需要明确授权。
2. fork + PR 流程需要 OAuth 用户身份。
3. 公开仓库读接口受 rate limit 影响。
4. 产品核心闭环应先建立在 GitHub App 已授权仓库上。

## 5. 后续验收标准

1. 未安装 GitHub App 时可以读取公开仓库 README。
2. 可以预览目标语言翻译内容。
3. 用户可以下载翻译 Markdown。
4. 点击提交 PR 时会引导授权。
5. 不会在未授权状态下写入任何仓库。
