# PRD 04: 目标语言选择

## 1. 目标

用户可以选择目标语言，系统根据语言代码生成目标文件后缀，并把语言要求传递给翻译 Provider。

## 2. 用户故事

作为用户，我希望从语言列表中选择目标语言，例如简体中文或英文，这样系统可以生成命名清晰的翻译文件。

## 3. 第一版语言列表

建议第一版内置：

1. `zh-CN`：简体中文。
2. `zh-TW`：繁体中文。
3. `en`：英语。
4. `ja`：日语。
5. `ko`：韩语。
6. `fr`：法语。
7. `de`：德语。
8. `es`：西班牙语。

## 4. 语言代码规则

1. 使用 BCP 47 风格语言代码。
2. 文件后缀直接使用语言代码。
3. `README.md` 翻译为 `README.zh-CN.md`。
4. `docs/guide.markdown` 翻译为 `docs/guide.zh-CN.markdown`。

## 5. 前端交互

语言选择组件需要：

1. 展示语言名称。
2. 展示语言代码。
3. 默认选择 `zh-CN`。
4. 支持搜索。
5. 明确展示输出文件预览。

## 6. 后端校验

后端必须校验：

1. 语言代码是否在允许列表。
2. 语言代码是否能用于文件名。
3. 源文件是否已经是目标语言版本。

不允许用户传入任意文件后缀，避免路径和命名风险。

## 7. 错误场景

1. 语言为空。
2. 语言不支持。
3. 语言代码格式非法。
4. 目标路径已存在。

目标路径已存在的策略：

1. 第一版默认更新目标语言文件。
2. PR 描述中说明这是更新已有翻译。
3. 不覆盖原始源语言文件。

## 8. 验收标准

1. 用户可以选择 `zh-CN`。
2. 系统能预览 `README.zh-CN.md`。
3. 不支持语言不能提交任务。
4. 目标路径已存在时能更新翻译文件。
