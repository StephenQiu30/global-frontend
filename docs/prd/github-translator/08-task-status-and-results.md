# PRD 08: 任务状态与结果展示

## 1. 目标

用户提交翻译任务后，前端必须展示任务执行状态、成功结果或失败原因，避免用户不知道系统是否正在工作。

## 2. 用户故事

作为用户，我希望提交翻译任务后看到进度和最终结果，这样我知道是否需要去 GitHub 审核 PR 或修复错误。

## 3. 状态页面

任务状态页展示：

1. 仓库名称。
2. 目标语言。
3. 选中文件。
4. 当前状态。
5. 成功 PR 链接。
6. 失败原因。
7. 重试入口。

## 4. 状态定义

1. `queued`：任务已创建。
2. `running`：任务执行中。
3. `succeeded`：PR 已创建。
4. `failed`：任务失败。

## 5. 结果展示

成功结果展示：

1. PR 标题。
2. PR URL。
3. 源文件与目标文件映射。
4. 翻译语言。
5. 下一步：去 GitHub 审核 PR。

失败结果展示：

1. 错误标题。
2. 错误详情。
3. 可能原因。
4. 建议操作。
5. 是否可重试。

## 6. 前端轮询

第一版可以使用简单轮询：

1. 每 2 秒查询一次任务状态。
2. 最多轮询 2 分钟。
3. 超过时间显示“任务仍在处理或已超时”。

如果后端采用同步执行，也可以提交后直接跳转结果页。

## 7. 验收标准

1. 用户提交任务后能看到处理中页面。
2. 成功时能看到 PR 链接。
3. 失败时能看到具体错误。
4. 用户可以从失败页返回修改输入。
5. 结果页展示源文件和目标文件映射。
