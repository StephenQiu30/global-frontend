# PRD 09: 安全、权限与滥用防护

## 1. 目标

系统必须确保只有被授权的仓库可以被翻译和提交，所有 GitHub 写操作都通过 GitHub App 权限执行，避免越权、路径穿越、误写文件和滥用模型额度。

## 2. 权限边界

第一版只支持：

1. 已安装 GitHub App 的仓库。
2. installation 授权范围内的仓库。
3. Markdown 文件翻译。
4. 通过 PR 提交。

第一版不允许：

1. 未授权仓库写入。
2. 任意路径写入。
3. 直接推送默认分支。
4. 删除文件。
5. 执行仓库代码。

## 3. 输入安全

必须校验：

1. 仓库地址格式。
2. owner/repo 合法性。
3. 文件路径不得包含 `..`。
4. 文件路径必须来自扫描结果。
5. 语言代码必须来自允许列表。
6. 文件扩展名必须为 `.md` 或 `.markdown`。

## 4. GitHub 安全

1. Webhook secret 必须配置。
2. GitHub App private key 必须只存在后端环境变量或 secret store。
3. installation token 不下发到前端。
4. 前端不能直接调用 GitHub 写接口。
5. 后端记录 GitHub API 错误但不泄露 token。

## 5. 模型调用安全

1. 限制单文件大小。
2. 限制单任务文件数量。
3. 限制总字符数。
4. Prompt 中明确不执行指令，只翻译文本。
5. 避免把 secrets 或非 Markdown 文件发送给模型。

## 6. 滥用防护

第一版建议：

1. 单任务最多 10 个文件。
2. 总原文最多 200KB。
3. 单 installation 简单频率限制。
4. 失败重试次数限制。

## 7. 验收标准

1. 未授权仓库不能提交任务。
2. 非 Markdown 文件不能提交任务。
3. 路径穿越输入会被拒绝。
4. installation token 不会出现在前端响应。
5. 所有写操作都只发生在翻译分支。
